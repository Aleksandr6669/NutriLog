# Автоматическое переключение GROQ токенов

## 📋 Описание

Приложение NutriLog имеет встроенный механизм для автоматического переключения между несколькими GROQ API ключами при исчерпании лимитов запросов. Это позволяет обеспечить непрерывную работу приложения даже если один из токенов исчерпал месячный лимит на запросы.

## 🎯 Функциональность

- **Список из 5 токенов:** Поддержка до 5 различных GROQ API ключей
- **Автоматическое обнаружение лимитов:** Система детектирует ошибки rate limit (код HTTP 429 и текстовые сообщения)
- **Круговое переключение:** После использования всех 5 ключей цикл начинается заново
- **Персистентное хранилище:** Текущий индекс ключа сохраняется в SharedPreferences
- **Лог переключений:** Debug сообщения с информацией о переключениях

## 🔧 Конфигурация

### 1. Подготовка переменных окружения

Скопируйте `.env.example` в `.env`:
```bash
cp .env.example .env
```

### 2. Добавление GROQ ключей

Отредактируйте файл `.env` и добавьте ваши GROQ API ключи:

```env
# Основной ключ (для обратной совместимости)
GROQ_API_KEY=sk-xxxxxxxxxxxxxxxxxxxx

# Список из 5 ключей для автопереключения
GROQ_API_KEY_1=sk-xxxxxxxxxxxxxxxxxxxx
GROQ_API_KEY_2=sk-xxxxxxxxxxxxxxxxxxxx
GROQ_API_KEY_3=sk-xxxxxxxxxxxxxxxxxxxx
GROQ_API_KEY_4=sk-xxxxxxxxxxxxxxxxxxxx
GROQ_API_KEY_5=sk-xxxxxxxxxxxxxxxxxxxx
```

> **Важно:** Используйте разные ключи из разных аккаунтов GROQ для максимальной эффективности.

### 3. Получение GROQ API ключей

1. Перейдите на [Groq Console](https://console.groq.com)
2. Зарегистрируйтесь или войдите в свой аккаунт
3. Перейдите в раздел "API Keys"
4. Создайте новые ключи для каждого из 5 слотов

## 🔄 Как работает автопереключение

### Процесс переключения

```
1. Приложение инициирует запрос к Groq API используя текущий ключ
   ↓
2. Если получен ответ успешно (2xx) → выполнение завершено ✓
   ↓
3. Если получена ошибка rate limit (429 или "quota exceeded") → переключение
   ↓
4. Сохранить новый индекс ключа в SharedPreferences
   ↓
5. Повторить запрос со следующим ключом
   ↓
6. Если все ключи исчерпаны → показать пользователю ошибку
```

### Пример из кода

```dart
// При получении ошибки rate limit
if (_isRateLimitError(response) && !tokenSwitchAttempted) {
  await _switchToNextGroqToken();  // Переключение на следующий ключ
  tokenSwitchAttempted = true;
  apiKey = _resolveApiKey().trim(); // Получение нового ключа
  continue; // Повторное выполнение цикла с новым ключом
}
```

## 📊 Состояние переключения

### SharedPreferences ключи

- **`ai.groq.token.index.v1`** - текущий индекс активного ключа (0-4)

### Debug вывод

При переключении в консоль выводится:
```
[Groq Token Switch] Switched from index 0 to 1
```

## ⚠️ Обнаруживаемые ошибки лимитов

Система автоматически обнаруживает следующие типы ошибок:

1. **HTTP 429** - Too Many Requests
2. **Текстовые сигнатуры в ответе:**
   - "rate limit"
   - "rate_limit_exceeded"
   - "quota"
   - "limit exceeded"
   - "too many requests"
   - "exceeded"

## 💡 Рекомендации

### Оптимальное использование

1. **Разные аккаунты:** Используйте разные GROQ аккаунты для каждого ключа
2. **Мониторинг:** Регулярно проверяйте использование лимитов на каждом аккаунте
3. **Распределение:** Убедитесь, что лимиты распределены достаточно для вашего использования

### Расчет лимитов

Если GROQ предоставляет:
- **500 запросов/месяц** на один ключ
- **5 ключей** = **2500 запросов/месяц** в сумме

Примерная нагрузка:
- Анализ рецепта: 2-3 запроса
- Статистика: 1-2 запроса
- Активность: 1 запрос

При использовании 5-10 раз в день это может быть ~200 запросов в месяц.

## 🐛 Отладка

### Проверка текущего ключа

Добавьте debug вывод в `_getCurrentGroqToken()`:

```dart
String _getCurrentGroqToken() {
  for (final token in _groqTokens) {
    final normalized = token.trim();
    if (normalized.isNotEmpty) {
      debugPrint('[Groq] Using token: ${normalized.substring(0, 10)}...');
      return normalized;
    }
  }
  return '';
}
```

### Проверка индекса

```dart
final index = await _getCurrentTokenIndex();
debugPrint('Current token index: $index');
```

## 🔄 Миграция с одного ключа на пять

Если вы используете только `GROQ_API_KEY`:

1. Сохраните текущий ключ как `GROQ_API_KEY_1`
2. Создайте 4 новых ключа в Groq Console
3. Добавьте их в `.env` как `GROQ_API_KEY_2` - `GROQ_API_KEY_5`
4. Перезагрузите приложение

Старый `GROQ_API_KEY` останется в качестве fallback для обратной совместимости.

## 📝 Логирование переключений

flutter run

## ❓ Часто задаваемые вопросы

### Q: Что будет, если все 5 ключей исчерпаны?
A: Приложение покажет пользователю ошибку: "Все GROQ ключи исчерпаны"

### Q: Сохраняется ли индекс ключа после перезагрузки приложения?
A: Да, индекс сохраняется в SharedPreferences и восстанавливается при запуске

### Q: Могу ли я использовать 4 ключа вместо 5?
A: Да, просто не добавляйте 5-й ключ в `.env`. Система будет циклировать по 4 ключам.

### Q: Как проверить, какой ключ сейчас активен?
A: Посмотрите значение `ai.groq.token.index.v1` в SharedPreferences или добавьте debug вывод

## 📞 Поддержка

Если вы столкнулись с проблемами:

1. Проверьте, что все ключи корректно добавлены в `.env`
2. Убедитесь, что ключи действительны в Groq Console
3. Проверьте логи в Flutter DevTools
4. Создайте issue в репозитории с описанием проблемы
